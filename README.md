# 📁 1folder — Android TWA + PWA

> Your single inbox for everything shared from Android — links, notes, documents, images, audio — saved automatically to a local folder or cloud storage.

## Features

- **Web Share Target** — appears in Android's share menu for any app
- **Accepts everything** — URLs, text/notes, PDF, DOCX, PPTX, images, audio, video
- **Offline-first** — Service Worker caches the app shell; shares are stored in IndexedDB even offline
- **Local folder sync** — File System Access API writes files to a folder you choose, organized by type
- **Cloud sync** — OneDrive, Google Drive, or Dropbox (OAuth 2.0, configure your Client IDs)
- **TWA wrapper** — Ships as a real Android APK via Bubblewrap; no browser chrome, no install prompt
- **Dark theme** — OLED-friendly, mobile-optimized UI

## Quick Start

### 1. Configure your domain

Search for `YOUR_DOMAIN` and replace with your actual domain everywhere:

```bash
find . -type f \( -name "*.json" -o -name "*.js" -o -name "*.webmanifest" -o -name "*.conf" \) \
  -exec sed -i 's/YOUR_DOMAIN/yourdomain.com/g' {} +
```

### 2. Generate icons

```bash
python3 generate_icons.py
```

### 3. (Optional) Configure cloud OAuth

In `public/js/storage-cloud.js`, replace the placeholders in `CONFIG`:

```js
const CONFIG = {
  onedrive: { clientId: 'YOUR_ONEDRIVE_CLIENT_ID', ... },
  gdrive:   { clientId: 'YOUR_GDRIVE_CLIENT_ID',   ... },
  dropbox:  { clientId: 'YOUR_DROPBOX_CLIENT_ID',  ... }
};
```

Each provider's redirect URI must be `https://YOUR_DOMAIN/oauth/{provider}`.

## Deployment options

### Option A — Firebase Hosting (recommended, free tier)

```bash
npm install -g firebase-tools
firebase login
firebase init hosting    # select "public" as public dir; "yes" to SPA
./build.sh deploy
```

### Option B — Vercel / Netlify

1. Push the repo to GitHub.
2. Import in Vercel/Netlify, set **root** to `public/`.
3. Add a rewrite rule: `/*` → `/index.html` (both platforms do this automatically).

### Option C — Self-hosted Nginx

```bash
sudo cp -r public/ /var/www/1folder/
sudo cp nginx.conf /etc/nginx/sites-available/1folder
sudo ln -s /etc/nginx/sites-available/1folder /etc/nginx/sites-enabled/
sudo certbot --nginx -d yourdomain.com
sudo nginx -s reload
```

Upload `assetlinks.json`:
```bash
sudo mkdir -p /var/www/1folder/android-twa/.well-known/
sudo cp android-twa/.well-known/assetlinks.json /var/www/1folder/android-twa/.well-known/
```

## Building the Android APK (TWA)

### Prerequisites

- Node.js 18+
- Java JDK 17+ (`java -version`)
- Android SDK (or Android Studio)
- `ANDROID_SDK_ROOT` env var set

### Steps

```bash
# 1. Install Bubblewrap
npm install -g @bubblewrap/cli

# 2. Initialize the TWA project (reads your live manifest)
./build.sh twa-init

# 3. Build debug APK
./build.sh twa-build

# 4. Install on device
adb install android-twa/app-debug.apk
```

### Release build (Google Play)

```bash
# 1. Build signed release AAB
./build.sh twa-release

# 2. Get SHA-256 fingerprint for assetlinks.json
./build.sh assetlinks
# → Paste into android-twa/.well-known/assetlinks.json
# → Upload/redeploy to https://yourdomain.com/.well-known/assetlinks.json

# 3. Upload .aab to Google Play Console
```

> **Why assetlinks.json matters**: Without it the TWA shows a browser toolbar.
> It proves you own both the domain and the APK signing key.

## How the share flow works

```
User taps "Share" in any Android app
        │
        ▼
Android resolves share_target from manifest.webmanifest
        │
        ▼
POST /share-handler  (multipart/form-data)
        │
        ▼
Service Worker intercepts → parses FormData
        │
        ├── text / URL  → entry {type: 'url'|'note', text, ...}
        └── files       → entry {type: 'images'|'docs'|'voice', content: ArrayBuffer, ...}
        │
        ▼
Saved to IndexedDB  →  App UI updated via postMessage
        │
        ▼
On sync (FAB / auto-sync):
  storageMode === 'local'    → File System Access API → chosen folder
  storageMode === 'onedrive' → MS Graph API  PUT /me/drive/root:/1folder/...
  storageMode === 'gdrive'   → Drive API     multipart upload
  storageMode === 'dropbox'  → DBX API v2    /files/upload
```
